## Homework Structure

Every lesson includes homework with 3 levels:

| Level | Description | Number of Exercises |
|-------|-------------|-------------------|
| 🟢 Easy | Practice the basics from the lesson | 5–7 exercises |
| 🟡 Medium | Apply concepts in new situations | 5–7 exercises |
| 🔴 Hard | Problem solving — real-world story problems | 5–7 exercises |

### Exercise Format

Every exercise clearly states:
- **Problem description** (what to do)
- **Input** (what the program receives, with constraints)
- **Output** (what the program should print)
- **Example** (sample input → sample output)

Hard exercises follow a **problem-solving format** inspired by competitive programming for kids:
a real-world story, clear input/output specification, and example test cases.

Answers are provided in a separate `answers.md` file for each lesson.

## Notes for Teachers

- Each lesson is 75 minutes.
- Students type slowly — allow extra time for coding.
- Use Thonny's variable inspector to help students visualize.
- Connect every new concept back to Scratch when possible.
- Encourage students to experiment and make mistakes.
